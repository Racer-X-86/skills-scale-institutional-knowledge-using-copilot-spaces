# OctoAcme Project Management Docs

Welcome to the central documentation site for OctoAcme's project management processes. These docs provide guidance, templates, checklists, and best practices for running projects from initiation through retrospective.

## Project Management Process Summary

OctoAcme's project management approach emphasizes:

- **Clear Ownership**: Each project has a named Project Manager and Product Lead who coordinate delivery and define outcomes.
- **Lifecycle Stages**: Projects progress through initiation, planning, execution, release, and retrospective phases, each with defined activities and deliverables.
- **Iterative Delivery**: Work is broken into small, testable increments with regular demos and feedback loops.
- **Risk Management**: Risks and dependencies are identified early, monitored continuously, and escalated appropriately.
- **Stakeholder Alignment**: Regular communication and status updates ensure all stakeholders are informed and aligned.
- **Data-Informed Decisions**: Projects measure impact against success metrics and iterate based on evidence.
- **Continuous Improvement**: Retrospectives capture learnings and convert them into actionable improvements.

### Core Roles

- **Project Manager**: Coordinates delivery, manages schedules, risks, and communications
- **Product Manager**: Defines outcomes, prioritizes the backlog, and measures success
- **Developers**: Implement features, collaborate on design and testability
- **QA/Testing**: Validate quality and acceptance criteria
- **Stakeholders**: Provide inputs, approvals, and business context
